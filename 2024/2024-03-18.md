22:24 DJDevon3
* Made some progress investigating a bug with the Adafruit ATECC608 crypto module. The serial number gets randomly truncated which causes the ensuing certificate process to fail with a CRC Mismatch error. By deleting the serial number functions and error handling I was able to produce a public certificate but with the serial number all zeros. That is actually great progress. One of the neatest features about this chip is that it can avoid an I2C scan as part of its security. Most people will believe they‚Äôve bricked the chip when they can no longer detect the I2C device and give up on using it, especially since it has a bug in Circuit Python that currently makes it unusable anyway. It will return the error ‚ÄúNo such device‚Äù. You must send it a specific frequency ping to wake it up and then access the I2C address. Sneaky little chip. I figured that out after hundreds of iterations and refused to believe it was dead.
